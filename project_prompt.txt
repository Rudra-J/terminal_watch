# Project Specification (Final+): Autonomous Claude Command Guard with Validation & Retry

## 1. Overview

This system evaluates and executes Claude-proposed commands using:

* Rule-based safety checks
* Local LLM (Ollama)
* **Strict output validation**
* **Retry + recovery logic**

It must be **robust against malformed LLM outputs and transient failures**.

## 3. LLM Output Contract (STRICT)

Expected JSON:

```json id="schema1"
{
  "decision": "ALLOW | DENY | ASK",
  "confidence": 0.0-1.0,
  "risk_level": "LOW | MEDIUM | HIGH",
  "reason": "string"
}
```

---

## 4. Validation Layer (Critical)

### 4.1 Schema Validation

Validate:

* JSON is parseable
* Required keys exist
* Types are correct:

  * decision → string (enum)
  * confidence → float ∈ [0,1]
  * risk_level → enum
  * reason → non-empty string

---

### 4.2 Semantic Validation

Reject outputs where:

* confidence is outside [0,1]
* decision not in allowed set
* empty or vague reason (e.g., "seems fine")
* inconsistent signals:

  * decision=ALLOW but risk=HIGH

---

### 4.3 Strict Parser

```python id="parser1"
def validate_output(obj):
    required = ["decision", "confidence", "risk_level", "reason"]
    
    for k in required:
        if k not in obj:
            return False
    
    if obj["decision"] not in ["ALLOW", "DENY", "ASK"]:
        return False
    
    if not (0 <= obj["confidence"] <= 1):
        return False
    
    if obj["risk_level"] not in ["LOW", "MEDIUM", "HIGH"]:
        return False
    
    if len(obj["reason"].strip()) < 5:
        return False
    
    return True
```

---

## 5. Retry Logic

### 5.1 Retry Conditions

Trigger retry if:

* JSON parsing fails
* Validation fails
* Empty response
* Timeout / API error

---

### 5.2 Retry Strategy

* Max retries: **3**
* Backoff: exponential

```python id="retry1"
for attempt in range(3):
    response = call_llm(prompt)
    
    try:
        parsed = json.loads(response)
    except:
        continue
    
    if validate_output(parsed):
        return parsed
    
    prompt = reinforce_prompt(prompt, response)
```

---

### 5.3 Prompt Reinforcement (Key Idea)

On retry, make the model stricter:

```id="reinforce1"
Your previous output was invalid.

You MUST return STRICT JSON with:
- correct schema
- valid confidence (0-1)
- valid enums

DO NOT include explanations outside JSON.
```

---

## 6. Fallback Policy (Fail-Safe)

If all retries fail:

```python id="fallback1"
return {
    "decision": "DENY",
    "confidence": 0.0,
    "risk_level": "HIGH",
    "reason": "LLM output invalid after retries"
}
```

→ Guarantees safety

## 7. Decision Engine (Unchanged Core)

```python id="decision1"
def decide(result, config):
    if result["decision"] == "DENY":
        return "BLOCK"
    
    if result["confidence"] <= config.deny_threshold:
        return "BLOCK"
    
    if result["decision"] == "ALLOW" and result["confidence"] >= config.allow_threshold:
        return "EXECUTE"
    
    return "ASK"
```

# Extension: Command Logging + Temporal Audit System

## 1. Objective

Add a **persistent, structured logging system** that records all Claude-requested commands per repository, enabling:

* Historical traceability
* Pattern analysis
* Future **window-based anomaly detection (audit model)**

---

## 2. Core Concept

Shift from:

> Single-command safety

To:

> **Behavior over time**

Examples of risks only visible over time:

* Gradual data exfiltration
* Repeated file probing
* Suspicious command sequences
* Privilege escalation attempts spread across steps

---

## 7. Classification Layer (New)

Before logging, classify command:

```python id="classify1"
def classify_command(cmd):
    if "git" in cmd:
        return "GIT"
    if "pip" in cmd or "npm" in cmd:
        return "INSTALL"
    if "python" in cmd:
        return "PYTHON"
    if "rm" in cmd or "cp" in cmd:
        return "FILE_OPS"
    if "curl" in cmd or "wget" in cmd:
        return "NETWORK"
    return "OTHER"
```

This is critical for future modeling.
